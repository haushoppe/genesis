About the Ordinal Cubes

Intro for visitors
The world's smallest onchain gallery, visualized as a 3D cube. Permissionless. Immutable. Forever existing. Choose six images that are already on the 🟧 Bitcoin blockchain and claim them with your own cube!

Detailed Description
Ordinal Cubes is a permissionless art project, anyone can create an onchain gallery, displaying existing images from the Bitcoin blockchain. The images are visualised as a 3D cube, which looks cool and holds the viewer's attention for a longer time. The created ordinals use recursive inscriptions heavily, so the cost of an inscription for each cube is very affordable.

Pure ordpool-sdk minting

Testimonial (by Johannes)
I didn't want to create an art collection where all works are already known at the beginning. Therefore, it was my goal that all parameters for the mint can be entered by the collector — inspired by fx(params). For the project's initial launch, cubes were minted through a hosted inscription service, which is the de-facto standard for most "lazy mints".

That model requires trust in a third party — the inscription service creates an order for you, and after you pay for the order, they hopefully start the inscription. In line with the Bitcoin ethos of self-custody and trustlessness, the optimal approach is for my collectors to create the transaction on their own, using their own device and private key. Cubes-frontend now uses ordpool-sdk to build and broadcast commit + reveal transactions directly from any ordinals-aware wallet — no OrdinalsBot, no OrdinalsBot referral kickback, no intermediary.

How the mint works
The website detects installed ordinals wallets (Xverse, Leather, Unisat, OKX, Phantom, Oyl) automatically and offers each as a connect option. Once connected, the mint form loads UTXOs from the wallet's payment address and estimates commit + reveal fees per candidate UTXO. Click "Mint my cube!" — your wallet prompts you to sign the commit; the reveal is signed with an ephemeral key generated in-browser and broadcast right after. As a fun side effect, both transactions carry nLockTime=21, so every cube mint also mints two free CAT-21 cats.
